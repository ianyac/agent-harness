# Stage 1: The Loop — Implementation Plan (Lessons 1–2)

> **For agentic workers:** This plan is NOT executed by subagents. It is a
> teaching plan: the human learner writes all implementation code; the teacher
> (Claude) briefs concepts, provides the test code below, reviews diffs, and
> verifies. Implementation steps therefore specify contracts and acceptance
> criteria, not code — that omission is deliberate, not a placeholder.

**Goal:** A REPL chat harness whose conversation state, model interface, and
provider adapter boundaries are proven by deterministic tests — lessons 1–2 of
the curriculum in `docs/superpowers/specs/2026-07-03-agent-harness-design.md`.

**Architecture:** Lesson 1 is a single file (`main.py`) containing the message
type, a scripted fake model, and the turn loop. Lesson 2 extracts the
`harness/` package, defines the `LLMClient` interface, and puts the
learner-written Codex adapter behind it. The fake model becomes the permanent
test double in `tests/`.

**Tech Stack:** Python 3.12+, `uv`, `pytest`. HTTP client (`httpx`) only if the
Codex adapter needs it.

## Global Constraints

- Python 3.12+, managed with `uv`; tests run via `uv run pytest`.
- No framework that owns the control flow. Libraries that stay out of the loop
  are fine.
- Invariant 1: nothing outside `harness/llm.py` touches a provider's raw
  request/response format.
- Automated tests never call the real API by default; real-API contract tests
  run only with `RUN_CODEX_TESTS=1`.
- Teaching protocol per lesson: concept briefing → tests first (red) → learner
  implements (green) → teacher reviews → REPL smoke test → commit → tag
  `lesson-NN`.
- Commit messages end with `Co-Authored-By:` trailer per session convention.

## Lesson 1: The conversation is the state

**Concept to brief before Task 1:** A chat "session" is nothing but a growing
list of messages. The model is a *stateless* function `messages → next
message`; it remembers nothing. Everything a harness does — tools, permissions,
compaction — is bookkeeping around that one function call. The fake model makes
this visible: it records exactly what it was shown.

### Task 2: run_turn — the seed of the agent loop

**Files:**
- Modify: `main.py`
- Test: `tests/test_lesson01.py` (append)

**Interfaces:**
- Consumes: `Message`, `FakeLLM` from Task 1
- Produces: `run_turn(messages: list[Message], user_input: str, llm) ->
  Message` — appends the user message, calls `llm.complete(messages)`, appends
  and returns the reply. Mutates `messages` in place. (Lesson 4 grows this
  into the full agent loop.)

- [ ] **Step 1: Write the failing tests**

```python
# append to tests/test_lesson01.py
from main import run_turn


def test_run_turn_appends_user_and_assistant_messages():
    llm = FakeLLM(["hello there"])
    messages = []
    reply = run_turn(messages, "hi", llm)
    assert messages == [Message("user", "hi"), Message("assistant", "hello there")]
    assert reply == Message("assistant", "hello there")


def test_model_sees_full_history_each_turn():
    llm = FakeLLM(["a", "b"])
    messages = []
    run_turn(messages, "one", llm)
    run_turn(messages, "two", llm)
    assert llm.calls[1] == [
        Message("user", "one"),
        Message("assistant", "a"),
        Message("user", "two"),
    ]
```

Teaching note: `test_model_sees_full_history_each_turn` will fail against a
naive `FakeLLM` that stores the list it was handed, even if `run_turn` is
correct. Let the learner hit this; the diagnosis (aliasing — `calls` must
record a *copy*) is the lesson's hidden gotcha and previews why transcripts
are snapshots.

## Lesson 2: The provider adapter

**Concept to brief before Task 4:** The harness owns its message format;
providers are plugins. The `LLMClient` interface is the socket, adapters are
the plugs, and a *contract test* — one test suite every implementation must
pass — is what keeps them interchangeable. This boundary is also what will
make streaming (chunks assembling into the same `Message`) and other providers
addable without touching the loop.

### Task 5: LLMClient interface + contract test

**Files:**
- Create: `harness/llm.py`
- Test: `tests/test_llm_contract.py`

**Interfaces:**
- Consumes: `Message` from `harness.messages`, `FakeLLM` from `tests.fake_llm`
- Produces: `LLMClient` Protocol in `harness/llm.py`:
  `def complete(self, messages: list[Message]) -> Message: ...`
  Every current and future client (FakeLLM, CodexAdapter) must satisfy the
  contract test below.

- [ ] **Step 1: Write the contract test (teacher explains parametrization)**

```python
# tests/test_llm_contract.py
import os

import pytest

from harness.messages import Message
from tests.fake_llm import FakeLLM


def available_clients():
    clients = [FakeLLM(["scripted reply"])]
    if os.environ.get("RUN_CODEX_TESTS") == "1":
        from harness.llm import CodexAdapter

        clients.append(CodexAdapter())
    return clients


@pytest.mark.parametrize("client", available_clients())
def test_complete_returns_a_nonempty_assistant_message(client):
    reply = client.complete([Message("user", "Reply with the single word: ping")])
    assert reply.role == "assistant"
    assert isinstance(reply.content, str) and reply.content


@pytest.mark.parametrize("client", available_clients())
def test_complete_does_not_mutate_the_input(client):
    messages = [Message("user", "Reply with the single word: ping")]
    snapshot = list(messages)
    client.complete(messages)
    assert messages == snapshot
```

### Task 6: The Codex adapter

**Files:**
- Modify: `harness/llm.py` (add `CodexAdapter`, `to_provider`, `from_provider`)
- Create: `tests/fixtures/codex_response.json` (captured real response)
- Test: `tests/test_codex_adapter.py`
- Modify: `main.py` (REPL uses `CodexAdapter` instead of `FakeLLM`)

**Interfaces:**
- Consumes: `Message`, `LLMClient`
- Produces:
  - `to_provider(messages: list[Message]) -> list[dict]` — pure translation to
    the provider's wire format
  - `from_provider(raw: dict) -> Message` — pure parse of the provider's raw
    response JSON
  - `CodexAdapter()` — `.complete()` = `to_provider` → HTTP/subprocess call →
    `from_provider`. Satisfies the contract test under `RUN_CODEX_TESTS=1`.

- [ ] **Step 1: Learner researches how to call the Codex subscription**

Options to evaluate (learner's call, teacher available for questions):
shelling out to the Codex CLI (`codex exec`), calling its app-server/proto
mode, or hitting the OpenAI-compatible endpoint with the subscription's
auth token. Deliverable of this step: one raw response captured to
`tests/fixtures/codex_response.json` by any one-off means (curl, scratch

- [ ] **Step 2: Write the failing translation tests**

The `to_provider` expectation below assumes an OpenAI-chat-style wire format

adjust the expected literal to match reality — the *structure* of the test
stands.

```python
# tests/test_codex_adapter.py
import json
from pathlib import Path

from harness.llm import from_provider, to_provider
from harness.messages import Message


def test_to_provider_maps_roles_and_content():
    messages = [Message("user", "hi"), Message("assistant", "hello")]
    assert to_provider(messages) == [
        {"role": "user", "content": "hi"},
        {"role": "assistant", "content": "hello"},
    ]


def test_from_provider_parses_a_real_captured_response():
    raw = json.loads(Path("tests/fixtures/codex_response.json").read_text())
    msg = from_provider(raw)
    assert msg.role == "assistant"
    assert isinstance(msg.content, str) and msg.content
```
