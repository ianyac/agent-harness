# Stage 1: The Loop — Implementation Plan (Lessons 1–2)

> **For agentic workers:** This plan is NOT executed by subagents. It is a
> teaching plan: the human learner writes all implementation code; the teacher
> (Claude) briefs concepts, provides the test code below, reviews diffs, and
> verifies. Implementation steps therefore specify contracts and acceptance
> criteria, not code — that omission is deliberate, not a placeholder.

**Goal:** A REPL chat harness holding real multi-turn conversations over the
learner's Codex subscription, then brought under deterministic tests — lessons
1–2 of `docs/superpowers/specs/2026-07-03-agent-harness-design.md`.

**Architecture:** Spike-and-stabilize. Lesson 1 is the spike: a single
`main.py` — messages list, real Codex call, REPL — verified manually, with one
raw response captured as a fixture. Lesson 2 stabilizes: extract
`harness/llm.py` (`LLMClient` Protocol, `CodexAdapter`, `normalize()`), write
`FakeLLM` as the permanent test double, and pin behavior with tests. Messages
are plain dicts in OpenAI chat format throughout.

**Tech Stack:** Python 3.12+, `uv`, `pytest`. HTTP client / `openai` SDK /
subprocess inside the adapter only, per the learner's Codex research.

## Global Constraints

- Python 3.12+, managed with `uv`; tests run via `uv run pytest`.
- No framework that owns the control flow. Libraries that stay out of the loop
  are fine.
- Invariant 1: the loop only ever sees plain OpenAI-format message dicts.
  Provider SDK objects, HTTP transport, and auth never escape `harness/llm.py`
  (enforced from Lesson 2 — the spike is exempt until stabilized).
- Automated tests never call the real API by default; real-API contract tests
  run only with `RUN_CODEX_TESTS=1`.
- Spiked code is not "done" until the following lesson brings it under test.
- Commit messages end with `Co-Authored-By:` trailer per session convention.

## Lesson 1: The conversation is the state (spike)

**Concept to brief before Task 1:** A chat "session" is nothing but a growing
list of messages. The model is a *stateless* function `messages → next
message`; it remembers nothing — every call replays the entire conversation.
Everything a harness does — tools, permissions, compaction — is bookkeeping
around that one function call. Proof, built today: the model "remembers" your
name only because your list contains it; restart the process and the memory is
gone, because the memory *is* the list.

## Lesson 2: Stabilize — the adapter boundary and the test double

**Concept to brief before Task 3:** The spike works, but nothing protects it.
Stabilizing means drawing the boundary: an `LLMClient` interface (the socket),
`CodexAdapter` (the plug), and `FakeLLM` (the test plug) — with a *contract
test* every plug must pass, so they're interchangeable. From here on, the
harness never cares who answers: `complete()` takes plain dicts, returns a
plain dict; SDK objects, HTTP, auth stay inside the adapter. Every future
lesson tests the harness by scripting the model's side of the dialogue through
FakeLLM. This boundary is also what makes streaming and other backends
addable without touching the loop.

### Task 4: FakeLLM + run_turn under test

**Files:**
- Create: `tests/fake_llm.py`
- Modify: `main.py` (extract `run_turn` from the REPL body)
- Test: `tests/test_lesson02.py`

**Interfaces:**
- Consumes: nothing new
- Produces:
  - `FakeLLM(script: list[str])` in `tests/fake_llm.py` —
    `.complete(messages: list[dict]) -> dict` returns
    `{"role": "assistant", "content": <next scripted string>}`;
    `.calls: list[list[dict]]` records the messages passed to each call
  - `run_turn(messages: list[dict], user_input: str, llm) -> dict` in
    `main.py` — appends the user message, calls `llm.complete(messages)`,
    appends and returns the reply. Mutates `messages` in place. (Lesson 4
    grows this into the full agent loop.)

- [ ] **Step 1: Write the failing tests**

```python
# tests/test_lesson02.py
from main import run_turn
from tests.fake_llm import FakeLLM


def test_fake_llm_returns_scripted_responses_in_order():
    llm = FakeLLM(["first", "second"])
    assert llm.complete([{"role": "user", "content": "hi"}]) == {
        "role": "assistant",
        "content": "first",
    }
    assert llm.complete([{"role": "user", "content": "again"}]) == {
        "role": "assistant",
        "content": "second",
    }


def test_fake_llm_records_what_it_was_shown():
    llm = FakeLLM(["ok"])
    llm.complete([{"role": "user", "content": "hi"}])
    assert llm.calls == [[{"role": "user", "content": "hi"}]]


def test_run_turn_appends_user_and_assistant_messages():
    llm = FakeLLM(["hello there"])
    messages = []
    reply = run_turn(messages, "hi", llm)
    assert messages == [
        {"role": "user", "content": "hi"},
        {"role": "assistant", "content": "hello there"},
    ]
    assert reply == {"role": "assistant", "content": "hello there"}


def test_model_sees_full_history_each_turn():
    llm = FakeLLM(["a", "b"])
    messages = []
    run_turn(messages, "one", llm)
    run_turn(messages, "two", llm)
    assert llm.calls[1] == [
        {"role": "user", "content": "one"},
        {"role": "assistant", "content": "a"},
        {"role": "user", "content": "two"},
    ]
```

Teaching note: `test_model_sees_full_history_each_turn` will fail against a
naive `FakeLLM` that stores the list it was handed, even if `run_turn` is
correct. Let the learner hit this; the diagnosis (aliasing — `calls` must
record a *copy*) is the lesson's hidden gotcha and previews why transcripts
are snapshots.

### Task 5: The contract test

**Files:**
- Test: `tests/test_llm_contract.py`

**Interfaces:**
- Consumes: `FakeLLM`, `CodexAdapter`
- Produces: the contract every present and future `LLMClient` must satisfy.

- [ ] **Step 1: Write the contract test (teacher explains parametrization)**

```python
# tests/test_llm_contract.py
import os

import pytest

from tests.fake_llm import FakeLLM


def available_clients():
    clients = [FakeLLM(["scripted reply"])]
    if os.environ.get("RUN_CODEX_TESTS") == "1":
        from harness.llm import CodexAdapter

        clients.append(CodexAdapter())
    return clients


@pytest.mark.parametrize("client", available_clients())
def test_complete_returns_a_nonempty_assistant_message(client):
    reply = client.complete(
        [{"role": "user", "content": "Reply with the single word: ping"}]
    )
    assert type(reply) is dict  # plain dict — SDK objects must not leak
    assert reply["role"] == "assistant"
    assert isinstance(reply["content"], str) and reply["content"]


@pytest.mark.parametrize("client", available_clients())
def test_complete_does_not_mutate_the_input(client):
    messages = [{"role": "user", "content": "Reply with the single word: ping"}]
    snapshot = [dict(m) for m in messages]
    client.complete(messages)
    assert messages == snapshot
```
